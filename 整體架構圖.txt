三、核心 struct 設計
1. Command line 結構
typedef struct {
    char** cliVarDefs;
    bool simulate;
    char** cliVarDefs;
    int cliVarCount;
    char** requestedTargets;
    int requestedTargetCount;
} CommandLineOptions;

責任
儲存：
--file
--simulate
command line 上的 var=value
command line 上指定的 targets

2. Variable 結構
typedef struct {
    char* name;
    char** values;
    int value_count;
} Variable;
責任：
表示一個最終變數，例如：
COMPILER = gcc
COMPILEFLAGS = -Wall -Wextra -pedantic
這裡直接存成 string array，不要只存 raw string。
原因：
後面組 command 時，spec 其實是把變數值視為「一串 strings」，不是一整條未切割字串。

3. Variable definition line
typedef struct {
    char* name;
    char** raw_tokens;
    int raw_token_count;
    int line_number;
} VariableDef;
責任：
表示 uqmakefile 中讀到的一條 variable definition line。
為什麼跟 Variable 分開
因為：
VariableDef 是 parse 出來的原始資料
Variable 是最後套用預設值 + file defs + CLI defs 之後的結果

4. Target type
typedef enum {
    TARGET_FILE,
    TARGET_PHONY
} TargetType;

5. File target
typedef struct {
    char* name;

    char** raw_dep_tokens;
    int raw_dep_count;

    char** expanded_deps;
    int expanded_dep_count;
    bool deps_expanded;

    int line_number;
} FileTarget;
責任
表示：
prog: main.o util.o
raw_dep_tokens：parse 階段存的原始 token
expanded_deps：build 時第一次展開後 cache 起來
deps_expanded：保證同一行只展開一次
這個 cache 很重要，因為 spec 要求 dependency / phony line 的 variable expansion 對同一行最多只做一次。

6. Phony target
typedef struct {
    char* name;

    char** raw_pipeline_tokens;
    int raw_pipeline_count;

    char** expanded_pipeline_tokens;
    int expanded_pipeline_count;
    bool pipeline_expanded;

    int line_number;
} PhonyTarget;
責任
表示：
clean ~ rm -f *.o
或包含 => 的 pipeline。

7. Unified target entry
typedef struct {
    TargetType type;
    char* name;
    union {
        FileTarget file_target;
        PhonyTarget phony_target;
    } data;
} Target;

8. Parsed makefile model
typedef struct {
    VariableDef* variable_defs;
    int variable_def_count;

    Target* targets;
    int target_count;

    char* first_targetName;
} UQMakefileModel;

責任
整份 uqmakefile 的 parse 結果。
包含
所有 variable definitions
所有 targets
第一個 target 名稱（command line 沒指定 target 時要用）

9. Variable table / environment
typedef struct {
    Variable* vars;
    int var_count;
} VariableTable;
責任
儲存最終變數值。
順序來源：
1. defaults
2. uqmakefile defs
3. CLI defs

10. Build result
typedef struct {
    bool success;
    bool built_something;
    bool targetExists_as_file;
    bool is_phony;
    struct timespec mtime;
} BuildResult;
責任
每次 recursive build target 後回傳資訊。
為什麼需要
你後面要判斷：
dependency 是否被 build
dependency 是否比 target 新
dependency 是否是 phony
沒有這個 struct，build_target_recursive() 很快就會變成一堆雜亂 out-params。

11. Runtime state
這個是 execution / signal 的核心。
typedef struct {
    bool interrupted;
    pid_t* running_pids;
    int running_count;
    int running_capacity;
} RuntimeState;
責任
記錄現在有哪些 child 還活著
SIGINT 時知道要對誰送 signal
阻止收到 SIGINT 後再跑新 command

四、function 分工

A. main.c
int main(int argc, char* argv[])
責任
主流程控制。
建議流程
int main(int argc, char* argv[]) {
    CommandLineOptions options = parse_command_line(argc, argv);
    FILE* fp = open_uqmakefile_or_exit(options.makefileName);

    validate_cli_variable_definitions_or_exit(&options);

    UQMakefileModel model = parse_uqmakefile_or_exit(fp, options.makefileName);
    fclose(fp);

    check_requested_targets_exist_or_exit(&model, &options, options.makefileName);

    VariableTable vars = build_variable_table(&model, &options);

    RuntimeState runtime = init_runtime_state();
    install_signal_handlers(&runtime);

    build_requested_targets_or_exit(&model, &vars, &options, &runtime);

    free_runtime_state(&runtime);
    free_variable_table(&vars);
    free_uqmakefile_model(&model);
    free_command_line_options(&options);
    return 0;
}

做 usage-level parsing。
負責內容

解析 --simulate
分辨後面的 var=value 和 target

option 重複

empty argument
失敗就印 usage 並 exit 1。

void validate_cli_variable_definitions_or_exit(const CommandLineOptions* options)
責任

C. parser.c
UQMakefileModel parse_uqmakefile_or_exit(FILE* fp, const char* filename)
責任
逐行 parse 整個 uqmakefile。
內部流程
fgets() 一行行讀
blank/comment line 跳過
lexer_tokenize_line()
看第二個 token 決定 line type
呼叫對應 handler
蒐集所有錯誤
parse 完後若有錯 → exit 4
若沒有任何 dependency/phoney target → exit 5。

責任

E. build.c
void build_requested_targets_or_exit(...)
責任
決定 top-level target 是哪些，然後逐個 build。
規則
若 command line 有 targets → 依序 build
否則 build first_targetName。

BuildResult build_target_recursive(...)
責任
A2 核心函式。
遞迴 build 一個 target。
大致流程
1. 若 interrupted，停止
2. 先看 target 是 phony / file / not explicit
3. 若 phony：
   - expand pipeline if needed
   - simulate or execute pipeline
   - return built_something = true
4. 若不是 explicit target 或 explicit 但無 dependencies：
   - 若以 .o 結尾 → implicit .c rule
   - 否則 target 必須是現成 file，不存在則 exit 7
5. 若是 explicit file target with deps：
   - expand deps if needed
   - build each dependency
   - should_rebuild?
   - 若要 rebuild：
       .o -> compile command
       else -> link command
   - 否則若是 top-level target，印 no action required
6. return BuildResult

bool is_object_file_target(const char* name)
責任
判斷是不是 .o

char* make_matching_c_filename(const char* obj_name)
責任
abc.o -> abc.c

BuildResult handle_implicit_object_rule(...)
責任
當 .o 沒有顯式 dependency line 時，視為依賴對應 .c。

bool should_rebuild_file_target(...)
責任
根據 spec 判斷 file target 是否需要 rebuild。
條件
target 不存在
任何 dependency 被 build
dependency 比 target 新
任何 dependency 是 phony。

char** make_compile_command(...)
責任
生成 object compile command：
$COMPILER -c $COMPILEFLAGS modified-filename

char** make_link_command(...)
責任
生成 executable link command：
$LINKER $COMPILEFLAGS -o target dependencies $LINKFLAGS
其中 dependencies 不包含 phony targets。

void print_command(char** argv, int argc)
責任
用單一空格印出 command。
simulate 跟 real execution 都會用到。

void expand_file_target_dependencies_if_needed(FileTarget* target, VariableTable* vars)
責任
只在第一次需要時展開 dependencies，並 cache 起來。

void expand_phony_pipeline_if_needed(PhonyTarget* target, VariableTable* vars)
責任
只在第一次需要時展開 pipeline，並 cache 起來。

F. exec.c
int execute_command(char** argv, RuntimeState* runtime, bool simulate)
責任
執行單一 compile/link command。
行為
先印 command
若 simulate → 直接回成功
否則 fork + exec
parent waitpid
return child result

int interpret_child_status(int status)
責任
把 waitpid() 的 status 轉成 spec 要的 exit semantics：
正常非零 → 原 exit code
signalled → 99
exec fail → 98（通常 child 中 _exit(EXEC_FAILURE_STATUS)）

void execute_command_or_abort(...)
責任
給 build engine 用。
若 command fail，就印：
uqmakelite: cannot build target "target" - aborting
然後照 spec exit。

`int execute_pipeline(char*** commands, int* argc_list, int cmd_count,
                     RuntimeState* runtime, bool simulate)`

責任
執行 phony target pipeline。
要做的事
validity check
印整條 pipeline
simulate 就直接成功
否則建 pipes
第一個 stdin 來自 /dev/null
最後一個 stdout 去 uqmakelite stdout
所有 stderr 去 uqmakelite stderr
全部 child 都是 uqmakelite direct children
reaping children
pipeline failure → exit 13。

bool is_valid_pipeline_after_expansion(...)
責任
檢查 command pipeline 在展開後是否合法。
不合法就印 invalid command pipeline，exit 12。

char*** split_pipeline_into_commands(...)
責任
把展開後 token list 依 => 切成多個 commands。

RuntimeState init_runtime_state(void)
責任
初始化 runtime 狀態。

void install_signal_handlers(RuntimeState* runtime)
責任
用 sigaction() 裝 SIGINT handler。
spec 也明確說不要用 signal()。

void record_running_pid(RuntimeState* runtime, pid_t pid)
void remove_running_pid(RuntimeState* runtime, pid_t pid)
責任
管理活著的 child pid 列表。

void handle_interrupt_shutdown(RuntimeState* runtime)
責任
真正做：
對 running children 發 SIGINT
等待
超過一秒還活著就 SIGKILL
最後印 interrupted message，exit 11。

void free_runtime_state(RuntimeState* runtime)
責任
釋放 runtime 狀態持有的資源。

六、build_target_recursive 的建議簽名
建議用這種形式：
BuildResult build_target_recursive(
        UQMakefileModel* model,
        VariableTable* vars,
        const char* targetName,
        bool isTopLevel,
        const CommandLineOptions* options,
        RuntimeState* runtime);
這題 build decision 會用到：
model lookup
variable expansion
simulate flag
signal runtime
把這些都明確傳進去，會比偷偷用 global 好很多。
